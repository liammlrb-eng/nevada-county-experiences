# Pitch Resources

Companion files for the Western Nevada County Experience pitch deck.
Drop this whole folder into Claude.ai chat (or share individual files)
to get strategic feedback on the audience, slide order, or talking points.

## Files in this folder

### Markdown (chat-friendly — copy/paste into Claude.ai)

| File | Purpose |
|---|---|
| `pitch_strategy_brief.md` | **Start here.** Paste this as the first message in a fresh Claude.ai chat. Briefs chat-Claude on the project + asks for a stakeholder map. |
| `deck_outline.md` | Slide-by-slide markdown outline of `demo_pitch.pptx` — chat reads this faster than the binary .pptx. |
| `project_summary.md` | One-page overview of the platform: scope, data sources, tech, status. |
| `audience_map_template.md` | Empty stakeholder-map template — fill in once chat-Claude returns its first cut. |

### Binary snapshots (for upload to Claude.ai or sharing)

| File | Source of truth |
|---|---|
| `demo_pitch.pptx` | Generated by `../generate_slides.py` (auto-synced on each run) |
| `demo_pitch.pdf` | Generated by `../generate_demo_pdf.py` (auto-synced on each run) |
| `operator_guide.pdf` | Generated by `../generate_operator_guide.py` (auto-synced on each run) |

The binary files are also in the project root; this folder keeps a copy
so you can drag-and-drop the whole `pitch_resources/` folder into a chat
window or email without picking files individually.

## Workflow

1. Open Claude.ai in a new chat.
2. Paste `pitch_strategy_brief.md` as the first message.
3. Optionally attach `deck_outline.md` so chat-Claude can reference specific slides.
4. Iterate on the stakeholder map, objections, deck order, and discovery questions.
5. Bring the strategic conclusions back to Claude Code in this repo to regenerate the deck.
